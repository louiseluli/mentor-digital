// 01 — contexto

O PROBLEMA: DESINFORMAÇÃO

  • Brasil está entre os países mais afetados por fake news no mundo.
  • WhatsApp e Telegram são os principais vetores de propagação.
  • Notícias falsas se espalham 6× mais rápido que as verdadeiras.
  • Agências de fact-check têm capacidade limitada — não cobrem tudo.
  • Utilizador médio não sabe como verificar antes de compartilhar.
73%

dos brasileiros já receberam notícias falsas pelo WhatsApp
(Reuters Institute Digital News Report, 2023)

💡 SOLUÇÃO PROPOSTA

Chatbot + plataforma web que guia o utilizador por uma análise multi-dimensional — linguística, fact-checks e cobertura midiática — antes de partilhar.

// 05 — scoring

PONTUAÇÃO DE RISCO MULTI-DIMENSIONAL

COM FACT-CHECKS DISPONÍVEIS

overall = linguistic × 0.25 + fc_signal × 0.65 + (1 − coverage) × 0.10

SEM FACT-CHECKS (só NLP)

overall = linguistic × 0.70 + (1 − coverage×0.30) × 0.30

CONFIANÇA DA ANÁLISE (0–1)

confidence = 0.30 + (0.40 if fc_data else 0) + coverage × 0.30

// 10 — fontes nacionais

VERIFICADORES BRASILEIROS — RSS

Por que RSS além da Google FC?

  • Google FC indexa artigos com dias de atraso
  • Feeds RSS têm os artigos mais recentes em tempo real
  • Cobre notícias virais antes da indexação
  • Sem custo de API — usa stdlib xml.etree
FEEDS = { "Aos Fatos": "https://www.aosfatos.org/noticias/feed/", "Agência Lupa": "https://agencialupa.com.br/feed/" } TIMEOUT = 8.0 # segundos # Match: ≥ 2 termos-chave no título+snippet # Cache Redis: TTL 1h

Pipeline de matching

① Extrair keywords (>3 chars, sem stopwords PT, max 8)
↓
② Fetch paralelo dos feeds (asyncio.gather, timeout 8s)
↓
③ Parse XML com ElementTree — título + description
↓
④ Score por número de keywords encontradas (mín. 2)
↓
⑤ Top 5 resultados → cache Redis TTL 1h

Nota: Projeto Comprova

RSS activo mas apenas 3 artigos de 2019 — descartado por inactividade.
